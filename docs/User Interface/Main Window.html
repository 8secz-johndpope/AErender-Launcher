AErender Launcher Help User Interface

Main Window

All AErender Launcher UI elements correspond to specific aerender flags.

  • Project file path - correspond to -project flag. You can select your project by pressing on a Select File... button or by dropping project directly onto path field.
  • Output file path - correspond to -output flag. aerender will output video to selected directory, with the file name you typed + predefined file name or, if you leaved Default as file name, aerender will use just predefined file name from selected Output Module.
  • Output Module Preset - correspond to -OMtemplate flag. This defines lossless ouptput module and specific predefined file name mask. aerender will automatically fill specific values into that mask.
    Example: Output Module: Photoshop
    Predefined mask: [compName]_[#####].[fileextension]
    Output name: Comp 1_00001.psd
  • Properties section
    • Render with missing files - correspond to -continueOnMissingFootage flag. This will determine will aerender continue render operation even if any source footage item is missing. Only log alert will be shown.
    • Play sound when render finishes - correspond to -sound flag. This will turn on and off sound when aerender finishes rendering.
    • Threaded rendering - correspond to -mp flag. This will tell aerender to create more rendering processes to render multiple frames simultaneously. Though, I haven't spotted any noticeable difference in render time whether this parameter is enabled or not.
    • Custom properties - if enabled, then text in field under that checkbox will be added to an executable. Use that field only if you need some custom flags that is not implemented in Launcher.
    • Cache usage limit and Memory usage limit - correspond to -mem_usage flag. This sliders will determine what percentage of your total After Effects cache size and available RAM aerender could use. Use this parameters when your render ended with an error, because of insufficient memory or memory overload.
  • Rendering section
    • Compostion
      • Single/Multipse composition switch - will turn Multiple composition rendering mode on/off.
      • Composition name field - correspond to -comp flag. This field is essential when you need to render specific composition from your project. If you want to render all available compositions from in to out, leave this field empty.
      • Multiple compositions grid - is a storage for all your composition's names. When Multiple composition rendering mode is turned on, grid becomes visible and Composition Name field hide. Type all your compositions names in grid. Note that when you type frames into Start Frame or End Frame fields, it will be applied to all your compositions.
    • Rendering
      • Start Frame field - correspond to -s flag. This field will determine the starting frame of the composition. This field could be empty (it will tell aerender to render composition from the start of range).
      • End Frame field - correspond to -e flag. This filed will determine the ending frame of the compostion. This field could be empty (it will tell aerender to render composition to the end of range). When Split Render rendering mode is enabled, this field must not be empty.
      • Split Render grid - will contain framespans that composition will be splitted into. As for now, contains only information and useful for Launcher itself.
